### 五大核心支柱 (The Five Pillars)
1.  **`fx_staff` (万物资源池)**:
    - 不再局限于“技师”，而是支持“人力、空间、设备”三种资源类型。
    - 支持跨店调度与动态提成配置。
2.  **`fx_services` (全行业项目库)**:
    - 乐高式结构，通过 `category` 切换行业模式。
    - 支持复杂的阶梯定价与多资源绑定逻辑。
3.  **`fx_events` (排期主表 - Master Schedule)**:
    - 仅作为“时间容器”，记录谁在什么时间占用了什么资源。
    - **Metadata (元数据)**: 核心杀手锏，使用 JSONB 存储行业特有数据（如车牌号、宠物种、病历号），无需修改数据库结构即可支持新行业。
4.  **`fx_event_items` (分润流水表 - Financial Ledger)**:
    - **财务真理来源**。每一笔交易拆解为原子级流水：`项目A + 员工甲 + 成交价 + 实时计算提成`。
    - 支持“一人多项”或“多人一项”的复杂结算。
5.  **`profiles` (全球服务护照)**:
    - 打破商户孤岛，实现全球客户偏好追踪（如过敏史、咖啡偏好等）在不同商户间的合规流转。
    - **G-ID 唯一识别**: 每一位全球客户拥有唯一的 G-ID，其偏好数据存储在 `fx_global_passports`。
    - **自动同步**: 在预约弹窗中选择会员后，系统会自动检索并同步其全球护照数据，并在结账时自动将 G-ID 绑定至预约记录。

### 超越巨头的核心基因
- **AI 预测性盈利引擎**: 不仅记录预约，更通过历史数据分析自动优化排期，引导高客单价项目，帮商户赚钱。
- **数据快照 (Immutable Snapshots)**: 结账瞬间锁定当时的价格与提成比例，确保财务数据十年后依然可审计、不因资源变更而紊乱。
- **RLS 安全隔离**: 在数据库底层通过 `merchant_id` 砌死隔离墙，确保全球商户数据绝对安全。

## 2. Interaction Features (交互功能)
### 预约状态机与流程控制 (Status Machine & Workflow)
- **请求式预约模型 (Request-to-Book)**:
    - **逻辑**: 客户发起的预约默认为 `pending` 状态，不自动占用技师排期。
    - **防骚扰机制**: 通过“待确认”状态过滤虚假/恶意预约，老板手动核实（回拨电话）后才转为正式预约。
- **状态流转**:
    - `pending` (待确认) -> `completed` (已结账/手动完成) -> `deleted` (软删除)。
- **实时同步 (Real-time Sync)**:
    - 集成 Supabase `postgres_changes` 频道。当客户提交预约或老板修改状态时，所有在线端（手机、平板）的日历会即时刷新，无需手动下拉。

### 数据库架构升级 (Database Schema)
- **用户档案系统 (Profiles)**:
    - **表结构**: 新增 `public.profiles` 表，独立存储用户的生日 (`birthday`)、会员等级 (`member_level`)、积分 (`points`) 及自定义昵称。
    - **自动化同步**: 实现了 PostgreSQL 触发器 (`handle_new_user`)。每当有新用户通过 OAuth 或邮箱注册，系统会自动在 `profiles` 表中初始化一条关联记录。
- **存量数据迁移**: 编写并执行了 SQL 脚本，将旧版备注中的 `[STATUS:COMPLETED]` 标签自动转化为标准化的 `status` 字段数据。

### 身份验证与安全增强 (Auth & Security)
- **Auth 漏洞修复**: 彻底移除了基于 `localStorage` 的简易登录校验，全面切换为 Supabase Session 强校验。
- **角色路由保护 (RBAC)**:
    - 实现了基于用户元数据 (`user_metadata`) 的权限分发。
    - 商家强制进入 `/merchant/onboarding`，普通用户进入 `/me`，未授权访问自动重定向至登录页。

- **24小时免登录 (Persistent Unlock)**:
    - **原理**: 引入 `localStorage` 存储解锁时间戳 (`calendar_unlock_time`)。
    - **逻辑**: 成功输入密码 `0428` 后记录当前时间。下次访问时，若距离上次解锁不足 24 小时，系统将自动跳过锁定界面，解决了频繁刷新网页需要重登的痛点。

### 弹窗与手势优化 (Modal & Gestures)
- **状态感知窗口路由**:
    - **新预约/未开始**: 自动组合显示“预约信息”+“项目选择”窗口。
    - **进行中 (未过半)**: 自动组合显示“预约信息”+“会员详情”窗口，方便查看顾客历史。
    - **已过半/已完成**: 自动切换为“服务信息”+“结账预览”窗口，实现快速收银。
- **滑动删除视觉修复 (Anti-Shrink)**:
    - **原理**: 引入 `overscroll-y-contain` 禁用浏览器的橡皮筋回弹效果。
    - **逻辑**: 在 `handleTouchMove` 中增加了顶部边界检测，当用户在窗口顶部向下滑动（准备触发删除）时，强制拦截浏览器默认滚动，解决了窗口底部出现的“缩动/抖动”视觉 Bug。
- **侧边栏手势**:
    - 在左侧菜单栏（Sidebar）增加了**向左滑动隐藏**的功能，提升了单手操作体验。

## 3. 核心功能概览 (Legacy Reference)
本项目是一个美甲店收银与预约管理系统，核心组件为 `Calendar.tsx`。
它集成了预约创建、编辑、结账、金额管理、多技师分配以及日历访问保护功能。

## 4. 关键技术实现 (Security & Logic)
### 版本控制与安全增强
- **强制版本管理**: 引入 `APP_VERSION` 常量（当前 `1.0.2`），配合 Supabase `settings` 表进行版本锁死。
- **数据库行级安全 (RLS)**: 
    - 统一在 `supabase/client.ts` 注入 `x-app-version` Header。
    - SQL 策略：`USING (current_setting('request.headers', true)::json->>'x-app-version' = '1.0.2')`。

### 金额管理与持久化 (Perfect Billing Logic)
- **架构**: 引入 `total_amount` 和 `billing_details` (JSONB) 存储。
- **金额优先级**: 手动总金额 > 技师手动金额 > 项目价格修改。
- **离职员工防护**: 通过更换 API Key 或在 RLS 中硬编码版本号封锁访问权限。
- **报表准确性**: `PerformanceReport.tsx` 增加了状态过滤逻辑，仅计算 `status: 'completed'` 的订单，排除了待确认和已删除数据对财务报表的干扰。

- **层级穿透**: 修复了日历锁定遮罩层无法屏蔽 Header 和 Grid 交互的问题。
- **重复底栏**: 修复了 `page.tsx` 自定义底栏与 `layout.tsx` 全局底栏冲突的问题。
- **金额计算**: 修复了已结账预约再次打开时金额翻倍和手动改价丢失的逻辑错误。

## 8. 授权与版权保护
- **多重安全防护机制**:
    - **版本锁死**: 通过 `APP_VERSION` 常量与后端安全策略深度绑定。非授权版本无法进行数据写入，有效防止代码被恶意篡改后继续使用。
    - **远程授权校验**: 集成了基于 Supabase 的远程授权检测。系统启动时会校验当前版本的合法性，未授权设备或版本将触发锁定状态。
    - **API 防御**: 统一注入自定义请求头，利用数据库层面的策略直接拦截非法请求。
