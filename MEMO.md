# 项目开发记忆 (MEMO.md)
最后更新日期: 2026-03-11

## 1. UI 与 视觉风格 (Visual Identity)
### 极致透明与科技感 (Glassmorphism & Cyberpunk)
- **背景系统**: 
    - 引入了全屏背景图 `wallhaven-eo68l8.jpg`，并配合 `fixed` 布局和 `z-index: -10` 确保全局覆盖。
    - 使用 `bg-zinc-950/40` 渐变遮罩层，在保持壁纸通透感的同时确保文字内容可读。
- **顶栏 (Header)**:
    - 实现了完全透明化。移除了原有的 `bg-zinc-950/60` 背景和 `backdrop-blur-3xl` 模糊效果。
    - 通过 `drop-shadow` 增强文字和图标在复杂背景下的辨识度。
- **按钮与分类**:
    - 所有分类按钮采用透明毛玻璃质感，背景设为 `bg-white/10` 或 `bg-black/10`。
    - 图标保留了色彩识别特征（如餐饮的 `text-orange-400`），但背景完全通透。
- **底部导航栏 (Bottom Navigation)**:
    - **极致透明**: 最终调整为 `bg-transparent`，完全移除背景色和模糊效果。
    - **视觉增强**: 
        - 选中项添加了 `bg-cyan-400` 的顶部指示光条，并配合 `blur-2xl` 的发光效果。
        - 导航图标使用 `drop-shadow-[0_2px_4px_rgba(0,0,0,0.8)]` 确保在全背景图下的清晰度。
    - **去重修复**: 从 `layout.tsx` 中移除了旧的 `BottomNav` 组件，解决了底部双导航栏重叠的问题。
- **日历视觉优化**:
    - **已结账状态增强**: 为已结账 (COMPLETED) 的预约块添加了对应员工颜色的边框线（如 FANG 为粉红色边框），解决了在半透明背景下辨识度不足的问题。

## 2. 交互功能 (Interaction Features)
### 导航与路由
- **底部导航激活**: 为底部导航栏的“首页”、“发现”、“我的”按键绑定了真实的 Next.js 路由跳转（`/`, `/explore`, `/me`）。
- **发现页滑动切换 (Swipe Navigation)**:
    - **手势支持**: 在 `ExplorePage` 中实现了 `onTouchStart`, `onTouchMove`, `onTouchEnd` 手势监听。
    - **逻辑**: 左右滑动屏幕可快速切换“全部”、“商家”、“景点”、“海边”、“公园”等分类。
    - **自动滚动**: 切换分类时，顶部的分类栏会通过 `scrollIntoView` 自动平滑滚动到当前选中项的中心。
    - **动画反馈**: 引入了 `animate-fade-in` (CSS 关键帧) 实现内容切换时的淡入位移效果。

### 时间选择器手势革命 (Time Picker Gesture Revolution)
- **手势连线选择**: 彻底废弃了传统的数字输入和确定按钮。用户通过手指从左至右“一笔画”连线数字即可设定时间。
- **三列布局**:
    - 第一列（小时十位）：数字顺序调整为 `2, 1, 0`，且 `2` 位于左上角起始位。
    - 第二列（小时个位）：0-9 数字。
    - 第三列（分钟）：00, 15, 30, 45 刻度。
- **松开即确定**: 采用 `onTouchEnd` 监听，手指离开屏幕瞬间自动保存时间并关闭界面，极大提升了平板操作效率。
- **防冲突锁定**: 当滑动起点位于时间选择器的第一列数字时，系统会自动锁定为“时间选择手势”，禁用全局滑屏逻辑。
- **极致简约 UI**: 移除了所有标题、预览文本和提示词，仅保留核心数字阵列。对比度提升至 `white/80` 以确保强光下的清晰度。

### 访问保护与持久化
- **24小时免登录 (Persistent Unlock)**:
    - **原理**: 引入 `localStorage` 存储解锁时间戳 (`calendar_unlock_time`)。
    - **逻辑**: 成功输入密码 `0428` 后记录当前时间。下次访问时，若距离上次解锁不足 24 小时，系统将自动跳过锁定界面，解决了频繁刷新网页需要重登的痛点。

## 3. 核心功能概览 (Legacy Reference)
本项目是一个美甲店收银与预约管理系统，核心组件为 `Calendar.tsx`。
它集成了预约创建、编辑、结账、金额管理、多技师分配以及日历访问保护功能。

## 4. 关键技术实现 (Security & Logic)
### 版本控制与安全增强
- **强制版本管理**: 引入 `APP_VERSION` 常量（当前 `1.0.2`），配合 Supabase `settings` 表进行版本锁死。
- **数据库行级安全 (RLS)**: 
    - 统一在 `supabase/client.ts` 注入 `x-app-version` Header。
    - SQL 策略：`USING (current_setting('request.headers', true)::json->>'x-app-version' = '1.0.2')`。

### 金额管理与持久化 (Perfect Billing Logic)
- **架构**: 引入 `total_amount` 和 `billing_details` (JSONB) 存储。
- **金额优先级**: 手动总金额 > 技师手动金额 > 项目价格修改。
- **离职员工防护**: 通过更换 API Key 或在 RLS 中硬编码版本号封锁访问权限。

## 5. 已修复的重大问题
- **层级穿透**: 修复了日历锁定遮罩层无法屏蔽 Header 和 Grid 交互的问题。
- **重复底栏**: 修复了 `page.tsx` 自定义底栏与 `layout.tsx` 全局底栏冲突的问题。
- **金额计算**: 修复了已结账预约再次打开时金额翻倍和手动改价丢失的逻辑错误。

## 6. 后续开发注意事项
- **滑动阈值**: 当前滑动切换分类的最小触发距离为 `50px`。
- **图标引用**: 底部导航栏需确保 `lucide-react` 图标已正确导入。
- **保持原子性**: 修改 `Calendar.tsx` 时需注意 `useEffect` 的依赖项，避免无限循环。
