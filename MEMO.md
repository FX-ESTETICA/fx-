# 项目开发记忆 (MEMO.md)
最后更新日期: 2026-03-09

## 1. 核心功能概览
本项目是一个美甲店收银与预约管理系统，核心组件为 `Calendar.tsx`。
它集成了预约创建、编辑、结账、金额管理、多技师分配以及日历访问保护功能。

## 2. 关键技术实现
### 日历锁定与密码保护 (Calendar Lock & Protection)
- **锁定状态管理**：通过 `isCalendarLocked` 状态控制，默认进入页面时为锁定状态。
- **解锁密码**：当前预设密码为 `0428`。
- **安全交互屏蔽**：
    - **全覆盖遮罩层**：在 `Calendar.tsx` 根容器最底层添加了 `z-[9999]` 的 `backdrop-blur-3xl` 遮罩层，确保完全覆盖 Header 和日历 Grid。
    - **交互禁用**：锁定状态下，底层组件（Header、Grid、Modals）的 `pointer-events` 被显式设为 `none`，且 `opacity` 为 `0`。
    - **弹窗保护**：即使在锁定前有打开的详情弹窗或员工管理弹窗，进入锁定状态时也会被强制隐藏。
- **视觉反馈**：错误密码会触发输入框红色高亮和抖动动画提示。

### 金额管理与持久化 (Perfect Billing Logic)
- **新数据库架构**：已引入 `total_amount` (数字) 和 `billing_details` (JSONB) 字段作为主存储。
- **金额级联覆盖规则**：
    1. **项目价格修改**：在结账预览中修改单个项目价格，会通过 `diff`增量自动同步更新到“分配技师”的金额和“总金额”。
    2. **技师金额修改**：手动调整技师金额后，会自动同步更新到“总金额”。
    3. **总金额修改**：最终手动修改的总金额具有最高优先级，且持久化保存。
- **billing_details 结构**：
    ```json
    {
      "items": [{"name": "项目名", "price": 10, "staffId": "技师ID"}],
      "staff": {"技师名": 10},
      "manualTotal": 10
    }
    ```
- **备注字段备份 (Legacy Fallback)**：为了向下兼容，新逻辑依然会将金额序列化为 `[项目名_AMT:金额_IDX:索引]` 并存入备注。

### 交互逻辑
- **itemKey 稳定性**：使用 `${event.id}-${itemName}-${idx}` 作为 key，确保在多预约合并结账时，每个项目的价格修改都能精准映射并持久化。
- **自动初始化**：打开结账预览时，系统会优先从 `billing_details` 加载，若无则从备注解析，最后才根据默认价计算。这确保了“即使不结账也能保存金额”。
- **手势操作**：支持右滑（Swipe Right）切换结账预览界面，再次右滑执行最终提交并关闭窗口。
- **已结账预约查看**：允许点击已标记为 `[STATUS:COMPLETED]` 的预约块，自动显示结账清单预览，并提供返回箭头切换回编辑模式。
- **UI 样式**：
    - 已结账预约：背景透明度 5% (`bg-opacity-[0.05]`)，保留白色边框，文字内容保持原样不淡化。
    - 预约块对齐：内容上下左右居中，移除了冗余的边距和容器层级。

## 3. 已修复的重大问题
- **层级穿透问题**：修复了日历锁定遮罩层无法完全屏蔽 Header 和 Grid 交互的问题（通过 `z-index: 9999` 和 `pointer-events-auto` 彻底封死）。
- **金额翻倍问题**：修复了打开已结账预约时，由于 `hasSavedAmounts` 判定不严谨导致金额从 46 变为 92 的错误。
- **重复累加问题**：修复了手动改价（如 30）后重新打开窗口变为 68 (30+38) 的逻辑冲突。
- **变量作用域错误**：修复了 `isAlreadyCompleted` 和 `escapedItem` 在 `handleSubmit` 和 `useEffect` 中因初始化顺序导致的运行时报错。

## 4. 后续开发注意事项
- **日历解锁**：密码硬编码在 `Calendar.tsx` 的 `handleUnlock` 函数中。
- **数据库同步**：在 `handleSubmit` 中，所有自定义金额都会被序列化并追加到 `备注` 字段中，确保刷新后能正确解析。
- **技师分配优先级**：颜色 -> `STAFF_SEQ` -> 显式绑定，系统会按此优先级自动识别每个服务项目的技师。
- **保持原子性**：修改 `Calendar.tsx` 时需注意 `useEffect` 的依赖项，避免无限循环或金额二次计算。
